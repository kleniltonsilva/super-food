═══════════════════════════════════════════════════════════════
  INSTALACAO (fazer apenas 1 vez)
═══════════════════════════════════════════════════════════════

  1. Instale Python 3.12 de https://www.python.org/downloads/
     IMPORTANTE: Marque "Add Python to PATH" na instalacao!

  2. Clique duas vezes em INSTALAR.bat
     (Ele instala dependencias e cria a impressora virtual)


═══════════════════════════════════════════════════════════════
  O QUE TEM NESTA PASTA
═══════════════════════════════════════════════════════════════

  INSTALAR.bat      → Instala tudo (Python deps + impressora virtual)
  DESINSTALAR.bat   → Remove a impressora virtual

  TUDO.bat          → Inicia tudo de uma vez (servidor+bridge+impressao)
  SERVIDOR.bat      → Liga a impressora virtual (TCP porta 9100)
  SIMULAR.bat       → Envia pedidos falsos pelo spooler do Windows
  BRIDGE.bat        → Interceptador — detecta pedidos de plataformas
  IMPRESSAO.bat     → Impressao — recebe pedidos do Derekh e imprime

  virtual_printer/  → Codigo da impressora termica virtual
  bridge_agent/     → Codigo do agente interceptador
  printer_agent/    → Codigo do agente de impressao


═══════════════════════════════════════════════════════════════
  COMO FUNCIONA (Resumo)
═══════════════════════════════════════════════════════════════

  A "Termica Virtual 80mm" e uma impressora que aparece no Windows
  como se fosse uma termica real de 80mm. Mas ao inves de imprimir
  em papel, ela envia os dados pela rede (TCP porta 9100) para o
  SERVIDOR.bat que roda no mesmo computador.

  Isso permite testar os agentes sem impressora fisica:

  ┌──────────────┐     ┌───────────────┐     ┌──────────────┐
  │ SIMULAR.bat  │────>│ Spooler       │────>│ SERVIDOR.bat │
  │ (pedidos)    │     │ Windows       │     │ (TCP 9100)   │
  └──────────────┘     │ "Termica      │     └──────────────┘
                       │  Virtual 80mm"│
                       └───────┬───────┘
                               │
                       ┌───────┴───────┐     ┌──────────────┐
                       │ BRIDGE.bat    │────>│ Backend      │
                       │ (intercepta!) │     │ (cria pedido)│
                       └───────────────┘     └──────┬───────┘
                                                    │ WebSocket
                                             ┌──────┴───────┐
                                             │ IMPRESSAO.bat│
                                             │ (imprime!)   │
                                             └──────────────┘


═══════════════════════════════════════════════════════════════
  JEITO FACIL — TUDO.bat
═══════════════════════════════════════════════════════════════

  Clique duas vezes em TUDO.bat para iniciar tudo automaticamente.
  Ele abre 3 janelas em sequencia:
    1. SERVIDOR (verde) — impressora virtual TCP 9100
    2. BRIDGE (roxa) — interceptador de pedidos
    3. IMPRESSAO (vermelha) — imprime pedidos do Derekh

  Na primeira vez, Bridge e Impressao pedem configuracao (login).
  Depois, basta rodar TUDO.bat novamente.


═══════════════════════════════════════════════════════════════
  INICIAR COM WINDOWS (auto-start)
═══════════════════════════════════════════════════════════════

  Tanto o Bridge quanto a Impressao podem iniciar automaticamente
  quando voce liga o computador. Para ativar:

  1. Abra BRIDGE.bat → na configuracao, marque "Iniciar com Windows"
  2. Abra IMPRESSAO.bat → na configuracao, marque "Iniciar com Windows"

  Depois de ativar, eles iniciam sozinhos ao ligar o PC.
  Para desativar, desmarque a opcao na configuracao.


═══════════════════════════════════════════════════════════════
  TESTE 1 — Teste Completo (Bridge + Impressao)
═══════════════════════════════════════════════════════════════

  1. Clique em TUDO.bat (abre 3 janelas)
  2. Configure Bridge e Impressao na primeira vez
  3. Clique em SIMULAR.bat → opcao 1 (4 pedidos de teste)
  4. Verifique:
     - SERVIDOR (verde): recibos ESC/POS decodificados
     - BRIDGE (roxa): "Parse OK" + "Pedido criado!"
     - IMPRESSAO (vermelha): "Recebido pedido para impressao"

  No painel admin (/admin):
  - Pedidos: 4 novos com origem "iFood (Bridge)", etc
  - Bridge Impressora: 4 interceptados, status "processado"


═══════════════════════════════════════════════════════════════
  TESTE 2 — Testar so a Impressao
═══════════════════════════════════════════════════════════════

  Janela 1: Clique em SERVIDOR.bat (deixe rodando)
  Janela 2: Clique em IMPRESSAO.bat (configure na primeira vez)
            Selecione "Termica Virtual 80mm" como impressora

  Depois, crie um pedido pelo site do restaurante.
  O Printer Agent recebe via WebSocket, formata em ESC/POS,
  e envia para a impressora virtual. O SERVIDOR exibe o recibo.


═══════════════════════════════════════════════════════════════
  CONFIGURACAO DO BRIDGE (primeira vez)
═══════════════════════════════════════════════════════════════

  Ao abrir BRIDGE.bat pela primeira vez, aparece uma janela pedindo:
  - URL do servidor: https://superfood-api.fly.dev
  - Email e senha do restaurante
  - Selecionar "Termica Virtual 80mm" na lista de impressoras
  - Marcar "Criar pedido automaticamente" para teste completo
  - Marcar "Iniciar com Windows" se quiser auto-start


═══════════════════════════════════════════════════════════════
  CONFIGURACAO DA IMPRESSAO (primeira vez)
═══════════════════════════════════════════════════════════════

  Ao abrir IMPRESSAO.bat pela primeira vez, aparece uma janela pedindo:
  - URL do servidor: wss://superfood-api.fly.dev
  - Email e senha do restaurante
  - Selecionar "Termica Virtual 80mm" como impressora geral
  - Marcar "Iniciar com Windows" se quiser auto-start


═══════════════════════════════════════════════════════════════
  PROBLEMAS COMUNS
═══════════════════════════════════════════════════════════════

  "python nao encontrado"
  → Instale Python e marque "Add Python to PATH"
  → Ou reinstale Python marcando essa opcao

  "pywin32 nao instalado"
  → Abra Prompt de Comando e digite: pip install pywin32

  "Impressora nao encontrada"
  → Rode INSTALAR.bat como Administrador
  → Ou: Painel de Controle > Impressoras > verifique se existe

  "Conexao recusada na porta 9100"
  → O SERVIDOR.bat precisa estar rodando primeiro!

  "Bridge nao detecta os pedidos"
  → Verifique se selecionou "Termica Virtual 80mm" na config
  → Verifique se SERVIDOR.bat esta rodando

  "Impressao nao recebe pedidos"
  → Verifique se a URL comeca com wss:// (nao https://)
  → Verifique se fez login com sucesso
